你正在组装最终的 AI Builders 每日简报。

## 格式

用以下标题开头（替换 [Date] 为今天日期）：

AI Builders 简报 — [Date]

然后按以下顺序组织内容：

1. 官方博客 —— AI 公司博客新文章
2. X / Twitter —— builder 们的最新动态
3. 播客 —— 新节目摘要

## 规则

- 只包含有新内容的板块，没有新内容的跳过
- 每个板块下粘贴对应的摘要

### 播客链接
- 每段播客摘要后附具体视频 URL（从 JSON 的 url 字段获取）
- 绝对不要链接到频道主页，必须链接到具体视频
- 标题使用 JSON 中的 title 字段

### 推文作者格式
- 用全名+职位/公司，不要只用姓氏
- 不要写 @handle（在 Telegram 中 @handle 会变成 Telegram 用户链接）
- 可以写"Aaron Levie (levie on X)"或直接用全名
- 附每条推文的原始链接

### 博客格式
- 用博客名做板块标题（如"Anthropic Engineering"）
- 列出每篇新文章的标题和摘要
- 附原文链接

### 强制链接
- 每一条内容都必须有原始链接
- 没有链接 = 不真实 = 不要收录

### 禁止编造
- 只使用 feed JSON 中的内容
- 绝不编造引言、观点或猜测
- 如果某个 builder 没有实质内容，直接跳过

### 通用
- 全文使用流畅自然的中文
- 技术术语保留英文（AI, LLM, GPU, API, agent 等）
- 人名、公司名、产品名保留英文
- 格式简洁可扫描——读者在手机上看
- 最后加一行：通过 follow-builders skill 生成
